# goldmark 렌더링 전 CJK 대응 마크다운 전처리

goldmark는 CommonMark 스펙을 엄격히 따르는데, 이로 인해 한국어 문서에서 두 가지 문제가 발생한다.

1. **단일 `~` 삭제선 오인식**: goldmark의 GFM 삭제선 확장이 `~~` 뿐 아니라 단일 `~`도 구분자로 인식한다. `80~90` 처럼 범위를 나타내는 관용적 표기에서 멀리 떨어진 다른 `~`와 짝이 맞으면 의도치 않은 `<del>` 태그가 만들어진다.

2. **닫는 구두점 뒤 강조 실패**: `**코루틴(coroutine)**을` 처럼 닫는 괄호 바로 뒤에 한글이 오면, CommonMark의 right-flanking 규칙(구두점 앞 닫힘 구분자는 뒤에 공백/구두점이 필요)을 충족하지 못해 `**...**`가 그대로 출력된다.

goldmark 파서를 직접 수정하는 대신, `preprocessMarkdown` 함수에서 두 가지를 처리한다: 단일 `~`를 `\~`로 이스케이프하고, 구두점 바로 뒤 강조 구분자 앞에 U+200B(Zero-Width Space)를 삽입한다. ZWS는 goldmark에서 공백도 구두점도 아닌 문자로 분류되어 right-flanking 조건을 통과시키면서 시각적으로는 완전히 투명하다. 코드 펜스와 인라인 코드 스팬 내부는 건너뛴다.

## Consequences

- goldmark를 업그레이드해도 전처리 로직은 별도 유지보수가 필요하다.
- ZWS가 렌더링된 HTML의 `<strong>` 내부에 포함되나 시각적·의미적 영향은 없다.
